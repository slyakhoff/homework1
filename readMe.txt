Данная программа получает данные с сайта avito.ru. Запускать имеет смысл только с ключами «gather» и «stats». Ключ «transform» не работает т.к. разбор текстового файла идет сразу при вполнении команды «gather». Команда «gather» работает в тестовом режиме т.е. запрашивает и разбирает всего одно объявление, записывает данные в файл «data1_test.csv» и выводит эти же данные в консоль. Это связано с тем, что на авито довольно хорошая система защиты от скрапинга и скраулинга. В бевом режиме я собирал данные через службу тор примерно по 90 объявлений (metod gather_process()).Каждое объявление запрашивалось с интервалом 10-20 сек. Задержка выбиралась случайным образом. По этой же причине собрал так мало данных (300 строк). Из объявления выбираются следующие параметры (id, марка, модель, цена, год выпуска, количество фото, пробег, количество владельцев, количество вимволов в коментарии). Нестоит запускать скрапинг в боевом режиме без использования прокси тор, т.к. ip хоста запустившего это код быстро отправляется «в баню».
Ключ «stats» выводит первые 10 строк «боевого» файла с данными (data1.csv) и основные статистики по данным этого файла.

Я думаю, что полученные мною данные могут быть использованы в машинном обучении следующим образом. Можно попробовать найти корреляцию этих данных с для выяснения (с некоторой вероятностью) подано ли объявление от собственника или «перекупа». Вероятно для улучшения точности определения потребуется NLP комментария продавца.
